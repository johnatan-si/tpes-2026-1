## 4. Exemplo de história de usuário
**US01**  
Como **recepcionista**, quero **registrar um novo atendimento**, para **organizar a fila de encaminhamento**.

**Critérios de aceitação:**
- deve ser possível cadastrar nome e dados principais;
- o sistema deve registrar data e hora;
- o atendimento deve aparecer em uma lista de acompanhamento.

---

## 5. Observações
- O backlog é dinâmico.
- Mudanças devem ser justificadas e registradas.
- Itens concluídos devem manter histórico.
