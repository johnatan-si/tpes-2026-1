## 2. Objetivo do trabalho

Desenvolver uma solução de software para **um dos problemas apresentados em sala de aula**, aplicando os conteúdos da disciplina ao longo do semestre.

O projeto deve evoluir por meio de **entregas incrementais**, articuladas com:
- processos de software;
- requisitos;
- modelagem;
- princípios de projeto;
- padrões de projeto;
- arquitetura de software;
- testes.

Além disso, o grupo deverá utilizar **Scrum** como abordagem de organização e acompanhamento do trabalho.

## 4. Fluxo de trabalho com Scrum

O projeto será conduzido com adaptações de Scrum para o contexto acadêmico.

### Papéis sugeridos
- **Product Owner (PO):** responsável por manter a visão do produto e priorizar o backlog.
- **Scrum Master:** responsável por organizar o fluxo, remover impedimentos e acompanhar as sprints.
- **Time de Desenvolvimento:** todos os integrantes contribuem para análise, modelagem, implementação e documentação.

> Em grupos pequenos, uma mesma pessoa pode acumular papéis, desde que isso seja registrado em `docs/02_scrum_e_organizacao_do_grupo.md`.

### Artefatos obrigatórios
- Visão do produto
- Product Backlog
- Sprint Backlog
- Registro de planejamento das sprints
- Registro de revisão/retrospectiva
- Documentação técnica incremental

## 6. Como usar este template

1. Faça um fork ou copie este repositório.
2. Renomeie o projeto no topo deste `README.md`.
3. Preencha os arquivos da pasta `docs`.
4. Use as issues do GitHub para registrar histórias, tarefas e bugs.
5. Atualize os arquivos da pasta `sprints` a cada ciclo.
6. Gere as entregas parciais a partir dos arquivos já preenchidos.
7. Mantenha o histórico de evolução do trabalho no GitHub.

## 7. Boas práticas esperadas

- commits frequentes e significativos;
- clareza nos nomes de arquivos e versões;
- documentação incremental;
- backlog priorizado;
- rastreabilidade entre problema, requisitos, modelagem, arquitetura e testes;
- evidências de revisão e evolução do produto.
